# Appwrite Collections Schema for DealPost

This document defines the database collections to create in Appwrite console for the DealPost application.

## Collection: users

**Collection ID:** `users`
**Database:** `dealpost`

### Attributes

| Attribute        | Type     | Size | Required | Default | Notes                                  |
| ---------------- | -------- | ---- | -------- | ------- | -------------------------------------- |
| userId           | String   | 255  | Yes      | -       | User ID (generated by Appwrite)        |
| email            | String   | 255  | Yes      | -       | Unique email, indexed                  |
| name             | String   | 255  | Yes      | -       | Full name                              |
| phone            | String   | 20   | No       | -       | Phone number                           |
| avatar           | String   | 255  | No       | -       | File ID from storage, profile picture  |
| bio              | String   | 500  | No       | -       | User biography                         |
| businessVerified | Boolean  | -    | No       | false   | Is user verified as business           |
| gstIn            | String   | 15   | No       | -       | GST registration number (if business)  |
| location         | JSON     | -    | No       | -       | geo-object: {lat: number, lng: number} |
| createdAt        | DateTime | -    | Auto     | -       | Creation timestamp                     |
| updatedAt        | DateTime | -    | Auto     | -       | Last update timestamp                  |

### Indexes

```
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_createdAt ON users(createdAt);
CREATE INDEX idx_users_businessVerified ON users(businessVerified);
```

### Permissions

```
Users can read & update only their own documents:
- role:authenticated + user:$userId (read, update)
- user:$userId (write, delete)

Public read for business profile info:
- role:any (read, with null values filtered)
```

## Collection: listings

**Collection ID:** `listings`
**Database:** `dealpost`

### Attributes

| Attribute      | Type          | Size | Required | Default  | Notes                                  |
| -------------- | ------------- | ---- | -------- | -------- | -------------------------------------- |
| listingId      | String        | 255  | Yes      | -        | Primary ID (Appwrite generated)        |
| title          | String        | 255  | Yes      | -        | Listing title                          |
| description    | String        | 2000 | Yes      | -        | Full description                       |
| specifications | JSON          | -    | No       | []       | Array of spec objects                  |
| price          | Integer       | -    | Yes      | -        | Price in paisa (₹ × 100) for precision |
| quantity       | Integer       | -    | No       | 1        | Available quantity                     |
| category       | String        | 100  | Yes      | -        | Category name, indexed                 |
| images         | Array(String) | -    | No       | []       | Array of file IDs from storage         |
| ownerId        | String        | 255  | Yes      | -        | User ID of lister, indexed             |
| location       | JSON          | -    | No       | -        | geo-object: {lat, lng, address}        |
| status         | String        | 50   | No       | "active" | active/sold/pending                    |
| viewCount      | Integer       | -    | No       | 0        | Number of views                        |
| createdAt      | DateTime      | -    | Auto     | -        | Listing creation time                  |
| updatedAt      | DateTime      | -    | Auto     | -        | Last update time                       |

### Indexes

```
CREATE INDEX idx_listings_ownerId ON listings(ownerId);
CREATE INDEX idx_listings_category ON listings(category);
CREATE INDEX idx_listings_status ON listings(status);
CREATE INDEX idx_listings_createdAt ON listings(createdAt);
CREATE INDEX idx_listings_price ON listings(price);
```

### Permissions

```
Anyone can read active listings:
- role:any (read - filtered to status='active')

Owner can update own listings:
- user:$ownerId (read, update, write, delete)

Indexed owner read for admin:
- role:admin (read, update, delete)
```

## Collection: messages

**Collection ID:** `messages`
**Database:** `dealpost`

### Attributes

| Attribute      | Type          | Size | Required | Default | Notes                           |
| -------------- | ------------- | ---- | -------- | ------- | ------------------------------- |
| messageId      | String        | 255  | Yes      | -       | Message ID (Appwrite generated) |
| conversationId | String        | 255  | Yes      | -       | Parent conversation ID, indexed |
| senderId       | String        | 255  | Yes      | -       | Sender user ID, indexed         |
| receiverId     | String        | 255  | Yes      | -       | Receiver user ID, indexed       |
| message        | String        | 4000 | Yes      | -       | Message content                 |
| attachments    | Array(String) | -    | No       | []      | Array of file IDs               |
| read           | Boolean       | -    | No       | false   | Message read status             |
| readAt         | DateTime      | -    | No       | -       | When message was read           |
| createdAt      | DateTime      | -    | Auto     | -       | Message timestamp               |

### Indexes

```
CREATE INDEX idx_messages_conversationId ON messages(conversationId);
CREATE INDEX idx_messages_senderId ON messages(senderId);
CREATE INDEX idx_messages_receiverId ON messages(receiverId);
CREATE INDEX idx_messages_createdAt ON messages(createdAt);
CREATE COMPOSITE INDEX idx_messages_convo_created ON messages(conversationId, createdAt DESC);
```

### Permissions

```
Only sender & receiver can read/write:
- user:$senderId (read, write, delete own - via senderId check)
- user:$receiverId (read, update read status)

Realtime subscriptions:
- Both users subscribed to messages in conversation
```

## Collection: notifications

**Collection ID:** `notifications`
**Database:** `dealpost`

### Attributes

| Attribute      | Type     | Size | Required | Default | Notes                                |
| -------------- | -------- | ---- | -------- | ------- | ------------------------------------ |
| notificationId | String   | 255  | Yes      | -       | Notification ID                      |
| userId         | String   | 255  | Yes      | -       | Target user ID, indexed              |
| type           | String   | 50   | Yes      | -       | message/like/new_listing/report      |
| title          | String   | 255  | Yes      | -       | Notification title                   |
| content        | String   | 500  | No       | -       | Notification body                    |
| relatedId      | String   | 255  | No       | -       | Related entity ID (userId/listingId) |
| read           | Boolean  | -    | No       | false   | Read status                          |
| actionUrl      | String   | 500  | No       | -       | Where to navigate on click           |
| createdAt      | DateTime | -    | Auto     | -       | Creation timestamp                   |

### Indexes

```
CREATE INDEX idx_notifications_userId ON notifications(userId);
CREATE INDEX idx_notifications_read ON notifications(read);
CREATE INDEX idx_notifications_createdAt ON notifications(createdAt DESC);
CREATE COMPOSITE INDEX idx_notifications_user_unread ON notifications(userId, read) WHERE read = false;
```

### Permissions

```
User can only read own notifications:
- user:$userId (read, update)

Cannot delete own notifications (kept for history):
- Admins can delete if needed

Realtime subscriptions:
- user:$userId subscribes to own notifications collection
```

## Creating Collections in Appwrite Console

### Step-by-Step:

1. **Login** to Appwrite Console: `https://api.dealpost.in`
2. **Select Project:** DealPost
3. **Go to:** Databases → dealpost (or create if not exists)
4. **Create Collection → Fill Details:**
   - Collection ID: (from list above)
   - Document Security: Enabled
   - Permissions: Define as per permissions above

5. **Add Attributes** for each collection (copy from table above)
   - Set Size, Required, Default, Validation as needed

6. **Add Indexes** for each collection (copy SQL commands, translate to UI)
   - Index Name: idx*[collection]*[field]
   - Fields: Select from dropdown
   - Unique: Mark if needed

7. **Set Permissions** after collection creation:
   - Document Security → Add roles/users
   - Configure read, write, update, delete

### Using CLI (Alternative - Faster):

```bash
# Install Appwrite CLI
npm install -g appwrite

# Login
appwrite login

# Create projects & collections from JSON config (see appwrite-collections.json)
appwrite deploy
```

## Migration from Existing Database

If you have existing data:

1. **Export** from current database (SQL/JSON)
2. **Transform** into Appwrite format:
   - Map IDs to Appwrite format
   - Convert timestamps
   - Convert arrays/JSON fields
3. **Import** via:
   - Appwrite API bulk insert
   - Custom Node.js script using Appwrite SDK
4. **Verify** data integrity before cutover

Example migration script provided in `migrate-to-appwrite.js`

## Security Considerations

✓ All collections have document-level permissions
✓ Users can only access own data
✓ Public listings visible only when status='active'
✓ Messages encrypted over HTTPS
✓ File uploads validated on backend
✓ Admin panel restricted to admins via role:admin
✓ Sensitive fields (passwords, tokens) NOT stored in collections

---

## Performance Tips

1. **Queries**: Always include indexed fields in WHERE clauses
2. **Pagination**: Limit to 100 docs per query, use offset
3. **Realtime**: Subscribe only to necessary collections/documents
4. **Caching**: Use browser/CloudFlare cache for public listings
5. **Batch Operations**: Use bulk API for many inserts (if available)

## Testing Collections

```javascript
// After creation, test via SDK:
const databases = new SDK.Databases(client);

// Create user
const user = await databases.createDocument(
	"dealpost", // database
	"users", // collection
	SDK.ID.unique(),

		userId: SDK.ID.unique(),
		email: "user@example.com",
		name: "John Doe",
	},

// Query listings
const listings = await databases.listDocuments("dealpost", "listings", [
```
