## One-Time Setup Commands

### Install Docker & Firewall

```bash
# System update
sudo apt update && sudo apt upgrade -y

# Install Docker
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh && rm get-docker.sh

# Verify Docker
docker --version
docker compose version

# Enable firewall
sudo ufw enable
sudo ufw allow 22 80 443/tcp
sudo ufw status

# Add user to docker group
sudo usermod -aG docker $USER && newgrp docker
```

### Generate Secure Values

```bash
# Run ONCE and save results
openssl rand -hex 32  # Copy to OPENSSL_KEY_V1
openssl rand -hex 32  # Copy to OPENSSL_KEY_V2
openssl rand -base64 32  # Copy to DB_PASSWORD
openssl rand -base64 32  # Copy to INFLUX_PASSWORD
```

### Deploy Appwrite

```bash
mkdir -p ~/appwrite && cd ~/appwrite

# Copy docker-compose.appwrite.yml from your machine:
# scp docker-compose.appwrite.yml root@148.230.66.192:~/appwrite/

# Create .env with values from "Generate Secure Values" above
cat > .env << 'EOF'
_APP_ENV=production
_APP_DOMAIN=api.dealpost.in
_APP_DOMAIN_TARGET=api.dealpost.in
_APP_OPENSSL_KEY_V1=<paste-from-above>
_APP_OPENSSL_KEY_V2=<paste-from-above>
DB_PASSWORD=<paste-from-above>
INFLUX_PASSWORD=<paste-from-above>
_APP_DB_HOST=mariadb
_APP_DB_PORT=3306
_APP_DB_USER=appwrite
_APP_DB_PASS=${DB_PASSWORD}
_APP_REDIS_HOST=redis
_APP_REDIS_PORT=6379
_APP_INFLUXDB_HOST=influxdb
_APP_INFLUXDB_PORT=8086
_APP_INFLUXDB_USERNAME=appwrite
_APP_INFLUXDB_PASSWORD=${INFLUX_PASSWORD}
_APP_STORAGE_DEVICE=local
_APP_LOGGING_LEVEL=error
_APP_SECURITY_COOKIE_SECURE=true
_APP_SECURITY_COOKIE_HTTPONLY=true
EOF

# Start Appwrite
docker compose up -d && sleep 60
docker ps  # Verify all containers running
```

### Install Nginx & SSL

```bash
# Install Nginx
sudo apt install -y nginx

# Copy nginx config from your machine:
# scp nginx.conf.appwrite root@148.230.66.192:/etc/nginx/sites-available/appwrite

# Enable and test
sudo ln -s /etc/nginx/sites-available/appwrite /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default 2>/dev/null || true
sudo nginx -t  # Should say "successful"
sudo systemctl restart nginx

# Get SSL certificate (requires domain DNS to be set up)
sudo apt install -y certbot python3-certbot-nginx
sudo certbot certonly --nginx -d api.dealpost.in \
  --non-interactive --agree-tos -m admin@dealpost.in

# Verify certificate
sudo certbot certificates
```


### Restart Services

```bash
docker compose restart                       # Restart everything
docker compose restart appwrite              # Just Appwrite
sudo systemctl restart nginx                 # Just Nginx
```

### View Logs

```bash
docker compose logs -f appwrite | grep error
docker compose logs mariadb | grep error
sudo tail -f /var/log/nginx/error.log
```

### Database Backup

```bash
docker exec appwrite-mariadb mysqldump -u appwrite -p appwrite \
  > ~/backups/backup_$(date +%Y%m%d_%H%M%S).sql
```

### Database Restore

```bash
docker exec -i appwrite-mariadb mysql -u appwrite -p appwrite \
  < ~/backups/backup_YYYYMMDD_HHMMSS.sql
```

### Keep These Safe ⚠️

- Admin email & password (created during first console access)
- Frontend API Key (from Appwrite Settings)
- Backend API Key (from Appwrite Settings)
- Database password (in .env)

- .env file contents
- API keys
- Database credentials
- SSH private keys

### Store Securely

- Password manager (Bitwarden, 1Password, etc.)
- Encrypted backup file
- Separate document from source code

## Troubleshooting Quick Fixes

| Problem                | Quick Fix                                     |
| ---------------------- | --------------------------------------------- |
| Containers not running | `docker compose restart`                      |
| DNS not resolving      | Wait 15 min, try `nslookup api.dealpost.in`   |
| HTTPS not working      | Check firewall `sudo ufw status`, check DNS   |
| Database errors        | Check MariaDB: `docker logs appwrite-mariadb` |
| High memory usage      | Check if indexes created, limit Docker memory |
| Slow response          | Add indexes, check `docker stats`             |
| Can't login to console | Clear cookies, try incognito mode             |

## Emergency Procedures

### If Something Goes Seriously Wrong

```bash
# 1. Stop everything
docker compose down

# 2. Check backup exists
ls -la ~/backups/

# 3. Restore database
docker compose up -d  # Brings up fresh containers
docker exec -i appwrite-mariadb mysql -u appwrite -p appwrite < ~/backups/latest.sql

# 4. Verify
docker ps
curl -I https://api.dealpost.in

# 5. Report issue & monitor logs
docker compose logs
```

### Complete VPS Evacuation

```bash
# Backup everything
docker exec appwrite-mariadb mysqldump -u appwrite -p appwrite > backup.sql
docker run --rm -v appwrite_redis:/data -v $(pwd):/backup \
  redis:latest redis-cli -p 6379 BGSAVE

# Download to local machine
scp root@148.230.66.192:~/appwrite/backup.sql ./
scp root@148.230.66.192:~/appwrite/redis.rdb ./
```
